Build a single-page React app called CGPA Simulator. No backend, no authentication, no data persistence. All logic is frontend only.

The app uses the Nigerian 5.0 grading scale:
- 70–100 → A → 5.0
- 60–69 → B → 4.0
- 50–59 → C → 3.0
- 45–49 → D → 2.0
- 40–44 → E → 1.0
- 0–39 → F → 0.0

The CGPA formula is:
```
New CGPA = (Current CGPA × Total Units So Far + Semester Grade Points) / (Total Units So Far + Semester Units)
Semester Grade Points = Sum of (grade point × credit units) for each course
```

---

**Layout — three input cards, one result section:**

**Card 1 — Current standing**
- Current CGPA (number input, 0.00–5.00, step 0.01, required)
- Total units completed so far (number input, min 1, max 400, required)
- Small hint text under total units: "check your transcript"

**Card 2 — Next semester courses**
- Dynamic list of courses. Each row has:
  - Course name (text input, optional, placeholder "Course 1", "Course 2" etc.)
  - Credit units (number input, min 1, max 6, required)
  - Remove button (× circle) — cannot remove if only one row remains
- "Add course" button below the list, adds a new empty row
- Maximum 15 courses
- Start with 3 rows by default

**Card 3 — Target**
- Target CGPA (number input, 0.00–5.00, step 0.01, required)
- Hint text: "must be higher than your current CGPA"

**Calculate button** — full width, below the three cards

---

**Result section — shown only after calculate is clicked:**

Two metric cards side by side:
- "Min. GPA needed next semester" — the calculated minimum semester GPA
- "Projected CGPA if achieved" — what their CGPA becomes if they hit that GPA

Per-course breakdown table below the metric cards:
- Columns: Course name (or "Course N" if unnamed), Credit units, Min. grade, Min. score
- Min. grade and min. score are the same target applied equally across all courses
- Derive the minimum grade point needed per course from the required semester GPA
- Map that grade point back to a letter grade and minimum score using the 5.0 scale above

Contextual message below the table. Use these rules:
- If min semester GPA ≤ 2.5 → green message: "You're in a comfortable position. Stay consistent."
- If min semester GPA between 2.5 and 3.5 → blue message: "This is achievable with focused effort."
- If min semester GPA between 3.5 and 4.5 → amber message: "This will require strong performance across all courses."
- If min semester GPA between 4.5 and 5.0 → orange/warning message: "This is very difficult but mathematically possible. You need near-perfect scores."
- If min semester GPA > 5.0 → red message: "This target is not reachable this semester given your current standing. Consider adjusting your target."

Small text at the very bottom of the result section: "Want to track your full CGPA history? Try TrackMyGPA →" linking to https://trackmygpa.vercel.app

---

**Validation rules (show inline errors, do not calculate if any fail):**
- Current CGPA must be between 0.00 and 5.00
- Target CGPA must be between 0.01 and 5.00
- Target CGPA must be greater than current CGPA
- Total units must be a positive whole number
- Every course must have credit units filled in (name is optional)
- Credit units per course must be between 1 and 6
- At least one course must be added

**Do not use localStorage, sessionStorage, or any browser storage. Results live in React state only and reset on refresh.**

**Styling:** Clean, minimal, white cards, generous whitespace. Mobile responsive. Use Tailwind CSS utility classes only.
